不是聊天窗口，而是科研操作系统

SciFlow Pro 的核心不是“多一个 AI 对话框”，而是把 AI 放进科研流程，让数据、证据和输出始终在同一个上下文里。

PRIVATE BY DEFAULT

本地优先的研究资产

实验记录、样品库、文献标注、知识库和报告默认沉淀在你的工作台里，适合长期课题和真实实验数据管理。

EXTENSIBLE WORKBENCH

按需安装的领域能力

用户不需要一次性安装全部模块。插件库让材料、理论计算、生物、化学、数据科学和工程验证方向各取所需。

EVIDENCE FIRST

从过程到成果可回溯

表征分析、性能验证、图表拼版和写作输出都连接到项目上下文，减少“结果有了但证据散了”的问题。

AI 不是独立问答框，而是嵌入工作流

文献阅读、实验记录、数据分析和论文写作都能回到同一个课题上下文，让 AI 的建议有来源、有对象、有后续动作。

计划 课题规划、实验路线和阶段报告保持连续。
执行 样品、参数、表征、性能和结论可以互相追溯。
复盘 外部 Agent 和本地 HTTP API 可以围绕项目数据协同。

插件库不是附属功能，而是产品结构

基础软件负责项目、数据和写作底座；领域插件负责把不同学科的专业工具带进同一个工作台。

云端索引，本地缓存，下载校验

插件包放在对象存储与 CDN，软件内从设置中的插件库读取索引，下载 zip 后做 hash 校验并解压缓存。这样可以先发布基础 SciFlow，再逐步补齐不同学科的领域包。

产品 Tour：从课题到输出的连续工作台

用真实界面展示 SciFlow Pro 的核心路径：先组织课题，再沉淀证据，最后进入图表、写作和分发。

01 · RESEARCH COMMAND CENTER

研究看板把课题状态变成可扫描的操作面

项目不只是列表，而是有路线、任务、样品、实验和风险的工作台。用户进入首页就能知道当前项目推进到了哪里。

  • 课题进度、团队任务和低库存预警集中呈现
  • 项目上下文直接连接实验、文献和报告
  • 适合长期课题的日常复盘和状态检查
02 · PROJECT FLOW

实验路线、样品和阶段报告在同一条链上

围绕课题路线组织实验，而不是把记录散落在多个文档里。每一次合成、表征和验证都能回到原始目的。

  • 路线节点和实验记录互相引用
  • 样品库保留来源、批次和性能证据
  • 周报、月报和计划可从项目上下文生成
03 · LITERATURE AND KNOWLEDGE

文献不是资料夹，而是项目知识的一部分

文献检索、标注、知识库和写作引用进入同一个研究语境，让 AI 总结和后续写作有明确来源。

  • 文献检索和项目知识库联动
  • 关键结论可回连到 PDF 与项目记录
  • 写作阶段保留引用和证据来源
04 · FIGURES AND DATA

数据图表服务于论文和汇报，而不是停在文件夹里

图谱、指标和图表模板围绕项目沉淀，后续写作、汇报和对比分析可以复用同一批证据。

  • 图表中心集中管理可复用科研图
  • 表征、性能和统计图进入同一输出流程
  • 后续插件可接入 XRD、XPS、BET、RDE 等分析
05 · WRITING STUDIO

从研究证据直接进入论文、报告和申请书写作

写作工坊不只是文本编辑器，而是把项目上下文、文献、图表和实验结论带入写作流程。

  • 报告和论文段落基于项目上下文组织
  • 图表、文献和实验证据可在写作中回看
  • 适合周报、月报、论文草稿和基金申请材料
